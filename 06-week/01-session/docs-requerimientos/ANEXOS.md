## Instrucciones paso a paso de la entrega

### Checklist de calidad

- [x] El documento tiene portada, índice y control de versiones.  
- [x] El propósito y alcance del sistema están claramente definidos.  
- [x] El glosario incluye todas las siglas y términos técnicos.  
- [x] Supuestos y dependencias externas están listados.  
- [x] El diagrama de perspectiva del producto está presente y explicado.  
- [x] RF/RNF son medibles, no ambiguos y verificables.  
- [x] Se incluyen al menos 6 RF y 6 RNF con ID único.  
- [x] Los RNF contemplan seguridad, rendimiento, accesibilidad y disponibilidad.  
- [x] RF tienen prioridad definida (Alta, Media, Baja o MoSCoW).  
- [x] Cada RF tiene criterios de aceptación o casos de prueba asociados.  
- [x] Historias de usuario cumplen formato “Como…, quiero…, para…”.  
- [x] Historias cumplen INVEST y tienen criterios Gherkin.  
- [x] Casos de uso incluyen actores, precondiciones y postcondiciones.  
- [x] Casos de uso incluyen flujos alternos y reglas de negocio.  
- [x] El diagrama de casos de uso corresponde con los IDs descritos.  
- [x] La matriz de trazabilidad cubre todos los RF críticos.  
- [x] No hay RF, HU o UC huérfanos sin trazabilidad.  
- [x] Los casos de prueba incluyen pasos y resultados esperados.  
- [x] Se realizó revisión por pares y se aplicaron ajustes.
